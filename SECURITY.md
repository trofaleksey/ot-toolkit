# Security and privacy reporting

OT Toolkit is pre-release software and is not a patient-record system or a claim of HIPAA compliance.

Report suspected vulnerabilities through the repository's [private Security Advisory flow](https://github.com/trofaleksey/ot-toolkit/security/advisories/new). If that channel is unavailable, open only a detail-free issue titled "Request private security contact"; the owner must provide a monitored private channel before receiving details. A direct fallback address must be configured before TestFlight distribution.

Never include real patient or child names, photos, labels, screenshots, clinical narratives, credentials, or unsanitized logs. Use synthetic reproduction data and describe any potentially sensitive attachment before sending it.

Privacy mistakes, unintended data transmission, backup/deletion failures, permission overreach, exposed app-switcher content, and persistence data loss are treated as security-relevant issues.
